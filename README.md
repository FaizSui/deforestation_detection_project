# Satellite Land-Cover Classification & Deforestation Detection

A transfer-learning image classifier (ResNet50, fine-tuned on EuroSAT) applied to real
Sentinel-2 imagery to detect forest loss along the BR-364 corridor near Porto Velho,
Rondônia, Brazil, validated against Global Forest Watch's Hansen Global Forest Change
dataset. Runs end to end on a single consumer GPU from free, public data.

The full write-up is in [`paper/`](paper/) (assembled: `paper/manuscript.md` /
`paper/manuscript.docx`). Guide credit: @collegewith_hari on Instagram.

## Pipeline in brief

Train ResNet50 on EuroSAT by two-stage transfer learning (98.0% test accuracy) →
download two co-registered Sentinel-2 images of one Amazon study area (2018, 2024) →
classify both into 64×64 patches → flag high-confidence Forest→non-Forest transitions →
validate against Hansen Global Forest Change. Scripts are numbered in run order; each
documents itself in its docstring.

## Results

| Metric | Value |
|---|---|
| Classifier test accuracy (frozen backbone → fine-tuned) | 94.5% → **98.0%** |
| Forest coverage, study area, 2018 → 2024 | 42.5% → 30.9% |
| High-confidence deforestation detections | 24 patches (8.7% of 2018 forest area) |
| Precision vs. Global Forest Watch | 75.0% (18/24 confirmed) |
| Recall vs. Global Forest Watch (all / eligible cells) | 10.5% (18/172) / 16.4% (18/110) |

## Setup

```bash
# PyTorch with CUDA 13.0 (swap the index for your CUDA version, or omit for CPU)
pip install torch torchvision --index-url https://download.pytorch.org/whl/cu130
pip install -r requirements.txt
```

Sentinel-2 download (script 06) needs a free Copernicus Data Space Ecosystem account:
register at https://dataspace.copernicus.eu, create an OAuth client at
https://shapps.dataspace.copernicus.eu/dashboard/ (User Settings → OAuth clients), and
put the credentials in a `.env` file at the repo root (never committed; see
`.gitignore`):

```
CDSE_CLIENT_ID=your-client-id
CDSE_CLIENT_SECRET=your-client-secret
```

## Scripts

| # | Script | Purpose |
|---|---|---|
| 01 | `download_dataset.py` | Download EuroSAT-RGB (27,000 patches) from Zenodo |
| 02 | `explore_dataset.py` | Class-balance chart + sample-image grid |
| 03 | `train.py` | Stage 1: train final layer, frozen backbone (10 epochs) |
| 04 | `evaluate.py --model {stage1,finetuned}` | Test-set report + confusion matrix |
| 05 | `finetune.py` | Stage 2: unfreeze all layers, lr 1e-4, keep best epoch |
| 06 | `download_sentinel2.py` | Fetch two single-date, haze-screened Sentinel-2 L1C images |
| 07 | `classify_patches.py` | Tile each image into 64×64 patches; classify + record confidence |
| 08 | `change_detection.py` | Flag high-confidence Forest→non-Forest transitions |
| 09 | `validate_gfw.py` | Compare detections against Hansen Global Forest Change |
| 10 | `sensitivity_analysis.py` | Sweep confidence threshold 70–95% (paper Table 2) |
| 11 | `recall_denominators.py` | Decompose recall into all-cell vs. eligible-cell |
| 12 | `publication_figures.py` | Render the four manuscript figures at 300 DPI |
| 13 | `assemble_manuscript.py` | Concatenate `paper/*.md` → `manuscript.md`/`.docx` |

## Reproducing the paper's tables

```bash
# Table 1 (per-class test metrics) + Figure 1 confusion matrix.
python scripts/01_download_dataset.py     # once
python scripts/03_train.py                # Stage 1  -> models/resnet50_stage1.pth
python scripts/05_finetune.py             # Stage 2  -> models/resnet50_finetuned.pth
python scripts/04_evaluate.py --model finetuned   # prints Table 1

# Table 2 (confidence-threshold sensitivity). Requires the two Sentinel-2
# images (script 06, needs .env credentials) and the classified grids (07).
python scripts/06_download_sentinel2.py
python scripts/07_classify_patches.py
python scripts/09_validate_gfw.py         # downloads the Hansen tile (once)
python scripts/10_sensitivity_analysis.py # prints Table 2
```

All splits use random seed 42; the fine-tuned model reproduces to 98.0% test accuracy.

## Data and reproducibility notes

`data/`, `models/*.pth`, and `figures/` are not tracked (see `.gitignore`) — they are
large, binary, and fully regenerated by the scripts above. EuroSAT is redownloaded from
[Zenodo record 7711810](https://zenodo.org/records/7711810); the Hansen validation tile
is fetched from the public Google Cloud bucket by script 09. Full environment and
acquisition details are in [`paper/reproducibility.md`](paper/reproducibility.md).

## Known limitations

- **Conservative by design.** One label per 640×640 m patch means only near-total
  clearing is flagged; partial clearing inside majority-forest patches is missed, which
  is why recall against pixel-level (30 m) Global Forest Watch is far below precision.
- **Reference blind year.** Hansen v1.11 covers loss only through 2023, so clearing in
  2024 cannot be cross-validated yet.
- **Domain shift.** Trained on European EuroSAT, applied to tropical Amazon; the
  L1C-vs-L2A processing-level match is essential (see the paper's Discussion).
