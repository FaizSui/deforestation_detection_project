# Satellite Land-Cover Classification & Deforestation Detection

A transfer-learning image classifier (ResNet50, fine-tuned on EuroSAT) applied to real
Sentinel-2 imagery to detect forest loss along the BR-364 corridor near Porto Velho,
Rondônia, Brazil, validated against Global Forest Watch's Hansen Global Forest Change
dataset.

Based on the workflow described in "How to Build a Satellite Image Classifier and
Detect Deforestation Using Machine Learning" (guide credit: @collegewith_hari on
Instagram).

## Results

| Stage | Result |
|---|---|
| Classifier test accuracy (frozen backbone) | 94.48% |
| Classifier test accuracy (fine-tuned) | **98.00%** |
| Forest coverage, study area, 2018 → 2024 | 41.5% → 31.2% |
| High-confidence deforestation flags (2018→2024) | 23 patches (8.5% of 2018 forest area) |
| Precision vs. Global Forest Watch | 82.6% (19/23 confirmed) |
| Recall vs. Global Forest Watch | 11.3% (19/168 GFW-flagged patches caught) |

See `figures/` (regenerated by the scripts below) for the confusion matrices, land-cover
maps, and the model-vs-GFW comparison map.

## Pipeline

Scripts are numbered and meant to be run in order. Each one explains itself in its
docstring.

1. `01_download_dataset.py` — downloads the EuroSAT RGB dataset (27,000 labeled 64x64
   satellite patches, 10 land-cover classes) from Zenodo.
2. `02_explore_dataset.py` — class balance chart + sample image grid, for sanity
   checking the data before training.
3. `03_train.py` — transfer learning, stage 1: ResNet50 pretrained on ImageNet, frozen
   backbone, only the final layer trained (10 epochs).
4. `04_evaluate.py --model stage1` — test-set evaluation, classification report +
   confusion matrix.
5. `05_finetune.py` — stage 2: unfreezes the whole network, continues training at a
   10x lower learning rate, keeps the best epoch by validation accuracy.
6. `04_evaluate.py --model finetuned` — re-evaluate after fine-tuning.
7. `06_download_sentinel2.py` — downloads real Sentinel-2 L1C imagery for the study
   area for two time periods (2018, 2024) via the Copernicus Data Space Ecosystem API,
   automatically picking the single clearest (least cloudy, least hazy) date in each
   window rather than blending multiple dates.
8. `07_classify_patches.py` — slices each image into 64x64 patches and classifies every
   one, saving both the predicted class and the model's confidence per patch.
9. `08_change_detection.py` — flags Forest→other transitions between years, filtered to
   only high-confidence predictions in both years.
10. `09_validate_gfw.py` — downloads the relevant Hansen Global Forest Change tile
    (public data, no account needed) and compares our flagged patches against
    independently-reported tree cover loss.

## Setup

```bash
pip install torch torchvision --index-url https://download.pytorch.org/whl/cu130
pip install matplotlib scikit-learn seaborn requests python-dotenv rasterio
```

Sentinel-2 imagery (step 7) requires a free Copernicus Data Space Ecosystem account:

1. Register at https://dataspace.copernicus.eu
2. Create an OAuth client at https://shapps.dataspace.copernicus.eu/dashboard/#/ (User
   Settings → OAuth clients)
3. Create a `.env` file in the project root (never committed -- see `.gitignore`):
   ```
   CDSE_CLIENT_ID=your-client-id
   CDSE_CLIENT_SECRET=your-client-secret
   ```

## Notes on data and reproducibility

`data/`, `models/*.pth`, and `figures/` are not tracked in this repo (see
`.gitignore`) -- they're large, binary, and fully regenerated by running the scripts
above in order. This keeps the repo small and focused on the actual code.

## Known limitations

- The classifier is conservative: it only flags near-total patch conversions and misses
  partial clearing within otherwise-forested 640m x 640m patches, which is why recall
  against Global Forest Watch (pixel-level, 30m resolution) is much lower than
  precision. A finer patch size or a per-pixel segmentation approach would likely close
  this gap.
- The Hansen Global Forest Change dataset (v1.11) only covers loss through 2023, so any
  clearing that happened specifically in 2024 can't be cross-validated yet.
- The classifier was trained on EuroSAT (European landscapes) and applied to a tropical
  region; this is a genuine domain shift, though results suggest it still generalizes
  reasonably well once the L1C/L2A processing-level mismatch was corrected for.
